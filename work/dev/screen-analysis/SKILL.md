# 嵌入式GUI屏幕分析 Skill

## 概述

接收嵌入式（车机/工控/智能家居等）GUI屏幕截图，进行专业级界面分析，最终生成带交互逻辑的HTML原型。

## 指令流程

### Step 1: 图片分析与界面认知

分析用户提供的界面截图，输出以下内容：

- **界面类型判断**：这是哪种类型的界面？（主菜单、设置页、弹窗、列表、媒体播放、拨号盘、仪表盘等）
- **推测产品场景**：车机？工控？智能家居？手持设备？
- **分辨率推测**：根据布局比例推测可能的分辨率（例如 800×480、1024×600、1280×720 等常见嵌入式分辨率）
- **整体布局结构**：顶部状态栏、底部导航栏、左侧菜单栏、右侧内容区、中央区域、弹窗层等

**交互询问（至少问以下问题，根据上下文可补充）**：
- "请问这张界面的实际分辨率是多少？"
- "这个界面的设计风格有什么参考吗？（例如：扁平风、拟物风、Material Design、自定义风格等）"
- "这个界面所在的设备是什么类型？"

### Step 2: 推测逻辑链

基于界面元素，推测用户的交互逻辑与页面流转，输出：

- **页面层级关系**：当前页面位于导航中的位置
- **按钮/点击区域交互**：每个可点击元素点击后应该跳转到哪里、弹出什么
- **列表/滑动交互**：如果有列表、滑动视图，说明滚动方向和内容
- **状态切换**：Toggle、Checkbox、Slider 等控件的交互状态
- **输入交互**：文本输入框、数字选择器等的输入方式
- **异常与边界状态**：空状态、加载态、错误提示应该在哪里展示

输出格式示例：
```
┌─ 逻辑链 ─────────────────────────────────────────┐
│                                                     │
│  [Screen] 主界面 (MainScreen)                        │
│    ├── 点击「设置」→ 跳转 SettingScreen              │
│    ├── 点击「音乐」→ 跳转 MusicScreen                │
│    ├── 滑动列表 → 浏览更多应用                       │
│    └── 下拉状态栏 → 弹出快捷开关面板                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

如果界面逻辑有歧义或不清楚的地方，**必须向用户追问确认**，例如：
- "这个按钮点击后是跳转新页面还是弹窗？"
- "列表支持滑动删除吗？"
- "顶部的返回箭头点击后回到哪个页面？"

### Step 3: 控件列表（完整详细）

列出界面中**每一个控件**的详细信息，使用表格格式：

| 控件名 | 控件类型 | X | Y | Width | Height | 父对象 | 功能描述 |
|--------|---------|---|---|--------|--------|--------|---------|
| status_bar | 状态栏 | 0 | 0 | 800 | 40 | Screen | 显示时间/信号/电量 |
| btn_settings | 按钮 | 720 | 5 | 40 | 30 | status_bar | 设置快捷入口 |
| ... | ... | ... | ... | ... | ... | ... | ... |

**命名规范**：控件名采用有意义的英文命名，类型前缀大写 + 下划线 + 描述（小写），如 `Btn_save`、`Lbl_title`、`Img_icon`、`List_menu`。

**坐标说明**：以屏幕左上角为原点 (0,0)，X为水平坐标，Y为垂直坐标。

### Step 4: 切图资源管理

#### 4.1 询问切图
完成控件分析后，询问用户：
> "是否需要对这些控件进行切图？我可以帮你命名，并生成对应的交互HTML。如果有现成的切图文件，请上传；如果没有，我可以使用白膜占位。"

#### 4.2 切图命名规则
如果用户提供切图文件但没有符合规范的命名，自动按以下规则重命名：

**格式**：`类型前缀_描述名.扩展名`

| 控件类型 | 前缀 | 示例 |
|---------|------|------|
| Button | Btn_ | Btn_save.png, Btn_back.png |
| Label | Lbl_ | Lbl_title.png, Lbl_value.png |
| Image | Img_ | Img_logo.png, Img_bg.png |
| Icon | Icon_ | Icon_wifi.png, Icon_battery.png |
| List | List_ | List_menu.png |
| Input | Input_ | Input_search.png |
| Slider | Slider_ | Slider_volume.png |
| Checkbox | Check_ | Checkbox_enable.png |
| Switch | Switch_ | Switch_toggle.png |
| Status Bar | Status_ | Status_bar.png |
| Background | Bg_ | Bg_main.png |
| Popup | Popup_ | Popup_dialog.png |
| Card | Card_ | Card_item.png |
| Progress | Prog_ | Prog_loading.png |

扩展名保持用户原始文件的扩展名（除非用户主动要求更改）。

#### 4.3 切图缺失处理
- 如果用户有切图但未上传，询问："请上传对应控件的切图文件"
- 如果用户没有切图或选择使用白膜：使用白色占位方块模拟控件（带虚线边框和控件名标注）

### Step 5: 生成交互式HTML

生成一个完整的、可直接在浏览器打开的HTML文件。

#### HTML必须包含的特性：

1. **精确还原**：尺寸、位置、颜色（近似）与截图一致
2. **切图渲染**：
   - 有切图的控件 → 显示对应图片
   - 无切图的控件 → 显示白膜占位（带控件名标签、虚线边框、浅灰背景，鼠标悬浮时显示详细信息 tooltip）
3. **交互逻辑模拟**：
   - 可点击元素支持鼠标点击反馈（按下/弹起状态）
   - 按钮悬浮时显示 tooltip 说明点击后的行为
   - 点击可交互元素时在页面顶部或侧边显示交互日志（如 "点击「设置」→ 跳转设置页面"）
4. **控件信息浮层**：每个控件支持点击或悬浮时显示信息卡片，包含：
   - 控件名称
   - 控件类型
   - 位置尺寸 (X, Y, W, H)
   - 父对象
   - 功能描述
5. **布局切换**：支持显示/隐藏控件边界辅助线（debug overlay 模式）
6. **标尺工具**：页面左上角和顶部显示刻度标尺辅助线

#### HTML技术选型：
- 纯 HTML + CSS + JavaScript（无外部依赖）
- 使用 Canvas 或 Div 布局均可
- 自适应缩放，默认以截图原始分辨率为基准

#### HTML文件命名：
- 默认命名为 `screen_preview_{界面名}.html`
- 存放在当前工作目录

## 输出总结

在生成HTML后，输出一份总结：

```
═══════════════════════════════════════════
  屏幕分析完成
═══════════════════════════════════════════
  界面名称：    XXX
  分辨率：      W × H

  交互动作：    N 个
  HTML文件：    screen_preview_xxx.html
───────────────────────────────────────────
  建议/备注：    ...
═══════════════════════════════════════════
```

## 控件类型参考

| 类型 | 说明 |
|------|------|
| Button | 按钮，可点击触发动作 |
| Label / Text | 文本标签，纯展示 |
| Image / Img | 图片控件 |
| Icon | 小图标（通常20-48px） |
| List | 列表控件 |
| Input / EditText | 文本输入框 |
| Slider / SeekBar | 滑动条 |
| CheckBox | 复选框 |
| Switch | 开关 |
| ProgressBar | 进度条 |
| StatusBar | 状态栏 |
| NavBar | 导航栏 |
| Card | 卡片容器 |
| Popup / Dialog | 弹窗 |
| Container / Panel | 容器面板 |
| TabBar / TabView | 标签页 |
| PageIndicator | 页面指示器（圆点） |
